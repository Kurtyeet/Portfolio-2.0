Case Study

Question 1 Case Study

The system solves manual leave management — eliminating paper-based tracking, spreadsheet dependencies, and the inability to know real-time leave balances. It centralizes leave types, employee records, and leave requests into one platform where balances are calculated automatically based on approved requests. Employees can apply for leave, check balances, and view history, while supervisors can approve or reject requests with comments.

Question 2 Case Study

Because leave management involves multiple stakeholders with different permissions:

  • Employee Submit requests, view own history and balance
  • Supervisor Approve/reject team requests, view team balances
  • HR Admin Full user management, system-wide reports, balance adjustments
  • System Admin Complete system access and configuration

Without role separation, employees could approve their own leave or access sensitive HR data.

Question 3 Case Study

The audit log serves as a compliance and accountability layer:

  • Compliance Tracks all leave actions with timestamps for HR audits
  • Accountability Records who approved/rejected each request with comments
  • Debugging Captures old/new values for data change investigations
  • Security Logs IP addresses and user agents for suspicious activity

Without it, leave disputes would have no evidence trail and compliance audits would fail.
